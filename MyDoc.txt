  we need to get the API endpoints: https://en.wikipedia.org/w/api.php, and add our page name
  which is page=Playwright_(software), and to add parameters such as: action=parse, prop-sections
  and format=json so we can read it easily, so we got:
  "https://en.wikipedia.org/w/api.php?action=parse&page=Playwright_(software)&prop=sections&format=json"
    - I have found that the "Index" of that is "5" so we can get:
    https://en.wikipedia.org/w/api.php?action=parse&page=Playwright_(software)&prop=wikitext&section=5&format=json
        - The API returns raw wikitext which contains markers like ===heading===, 
        * and <ref>...</ref>. 
        These all need to be removed during normalization according to the assignment.
    
    For the API call in code, we used Playwright's IAPIRequestContext which is Playwright's built in tool for making HTTP requests,
    we passed it Page.APIRequest from the test. The response comes back as JSON so we navigated it level by level — parse → wikitext → * to get the actual text.
    For the normalization, we created a TextNormalizer class with 2 functions:
        - Normalize() — cleans both texts by removing the wikitext markers we found earlier (===heading===, <ref>...</ref>, [15] footnotes, * bullets),
          and then removes punctuation and lowercases everything so both texts are in the same format.
        - CountUniqueWords() — splits the cleaned text into words and puts them in a HashSet which automatically removes duplicates,
          then returns the count.

    we used HashSet is because it doesn't allow duplicate values.
